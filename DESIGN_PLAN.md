# 家教课程账单 - 现代化 UI 改造方案

## 一、当前设计分析

### 现有问题
| 方面 | 问题描述 |
|------|---------|
| **色彩方案** | 以 `#f5f5f5` 灰色背景为主，配色单调，缺乏品牌感和活力 |
| **视觉层次** | 卡片阴影单一（`shadowOpacity: 0.1`），层次感弱 |
| **排版** | 字体大小和粗细变化有限，标题和正文区分不够明显 |
| **动态效果** | 完全无动画，所有交互都是静态的 |
| **底部导航** | 只有简单的颜色变化，缺乏视觉吸引力 |
| **空状态** | 灰色图标 + 文字，过于平淡 |
| **表单/模态框** | 标准输入框样式，缺乏设计感 |
| **整体风格** | 偏向传统的 Material Design 初代风格，缺乏现代感 |

---

## 二、改造目标

打造一款**清新、现代、有温度**的家教账单管理应用，设计关键词：
- **轻量灵动**：毛玻璃效果、柔和渐变、流畅动画
- **温暖专业**：暖色调为主，传达教育的温暖与信赖
- **层次分明**：通过阴影层级、间距节奏、色彩对比营造空间感
- **微交互**：按钮按压反馈、列表项滑入、数字跳动等细腻动效

## 三、全局设计系统

### 3.1 色彩体系

```
主色调（Primary）：
  主色:      #5B8DEF  (柔和蓝紫)
  深色:      #3D6FD9
  浅色:      #E8F0FE

辅助色（Accent）：
  暖橙:      #FF8C6B  (用于CTA按钮、重要提示)
  薄荷绿:    #4ECDC4  (用于收入、已完成状态)
  暖黄:      #FFD93D  (用于待处理、提醒)

功能色：
  成功/已收:  #34C759
  警告/待收:  #FF9500
  错误/删除:  #FF3B6E

中性色：
  背景:       #F2F4F8 (浅灰蓝，替代纯灰)
  卡片:       #FFFFFF
  标题:       #1A1A2E
  正文:       #4A4A6A
  辅助文字:   #9A9AB0
  分割线:     #EEEEF2
```

### 3.2 排版规范

| 层级 | 字号 | 字重 | 用途 |
|------|------|------|------|
| H1 | 28px | Bold (700) | 页面大标题 |
| H2 | 22px | Bold (700) | 区块标题 |
| H3 | 18px | SemiBold (600) | 卡片标题 |
| Body | 15px | Regular (400) | 正文内容 |
| Caption | 13px | Medium (500) | 辅助说明 |
| Small | 11px | Medium (500) | 标签、角标 |

### 3.3 圆角与间距

- 卡片圆角：16px（大卡片）/ 12px（小卡片）
- 按钮圆角：12px（常规）/ 24px（胶囊形）
- 图标容器圆角：14px
- 页面内边距：20px
- 卡片间距：16px
- 元素间距：12px（常规）/ 8px（紧凑）

### 3.4 阴影层级

```typescript
// 细微阴影 - 普通卡片
shadowSubtle: {
  shadowColor: '#1A1A2E',
  shadowOffset: { width: 0, height: 2 },
  shadowOpacity: 0.06,
  shadowRadius: 8,
  elevation: 2,
}

// 标准阴影 - 可交互卡片
shadowStandard: {
  shadowColor: '#1A1A2E',
  shadowOffset: { width: 0, height: 4 },
  shadowOpacity: 0.08,
  shadowRadius: 16,
  elevation: 4,
}

// 悬浮阴影 - FAB按钮
shadowFloating: {
  shadowColor: '#5B8DEF',
  shadowOffset: { width: 0, height: 8 },
  shadowOpacity: 0.2,
  shadowRadius: 24,
  elevation: 8,
}
```

## 四、各页面改造详情

### 4.1 首页 (HomeScreen.tsx)

#### 改造要点：

**① 头部区域**
- 去掉纯色圆形头像，改为渐变色背景 + 首字母展示
- 问候语使用渐变文字效果（主色调渐变）
- 日期使用更轻的字重，放在问候语下方
- 整体头部增加上下内边距，留白更多

**② 概览卡片**
- 改为一大一小不对称布局（不是两个等分）
- 待收款卡片占据 55% 宽度，今日收入占 45%
- 卡片背景应用微妙渐变（e.g. 待收：`#FFF5F5` → `#FFF0F0`）
- 数值使用大号加粗字体，添加数字递增动画
- 添加装饰性背景图案（半透明圆形、波纹）

**③ 快捷操作**
- 从 2x2 网格改为水平滚动的药丸形按钮
- 每个按钮包含图标 + 文字，带有浅色背景
- 选中/按下时轻微缩放动画（scale: 0.95）

**④ 最近课程**
- 每行左侧添加彩色竖条标识（课程状态色）
- 已收/待收标签从圆角矩形改为圆角胶囊形，增加透明度层次
- 列表项点击时添加轻微的背景色变化反馈
- 金额显示使用等宽数字字体效果

**⑤ 空状态**
- 插图风格图标（使用多个 Ionicons 组合）
- 引导文字更加温暖友好
- 添加按钮使用渐变背景 + 阴影

### 4.2 学生管理页 (StudentScreen.tsx)

#### 改造要点：

**① 学生卡片**
- 左侧添加科目颜色标识条（不同科目不同颜色）
- 姓名首字母作为圆形头像展示
- 卡片右侧显示时薪，使用强调色
- 编辑/删除图标改为更圆润的图标按钮

**② 添加按钮 (FAB)**
- 使用主题渐变色（紫色 → 蓝色渐变）
- 添加脉冲动画（呼吸效果）吸引注意
- 按下时旋转 45° 过渡到关闭状态

**③ 模态框**
- 顶部添加圆角拖拽指示条（模拟原生感觉）
- 标题栏使用毛玻璃效果背景
- 输入框改为带图标的圆角卡片样式
- 保存按钮使用渐变色，移除纯色背景

**④ 空列表状态**
- 添加友好的引导插画
- "添加第一个学生"引导文案 + 按钮

### 4.3 课程记录页 (LessonScreen.tsx)

#### 改造要点：

**① 课程卡片**
- 重新设计卡片信息层级：
  - 顶部：学生姓名(大) + 日期(小) + 付款状态标签(右对齐)
  - 中间：分隔线 + 课时/金额两行信息，使用 icon + 文字
  - 底部：编辑/删除操作
- 待收款卡片左侧边框使用暖橙色
- 已收款卡片左侧边框使用薄荷绿
- 付款状态标签可点击切换，带缩放弹跳动画

**② 学生选择器**
- 从底部弹出改为居中卡片式弹出（已有类似实现）
- 每个学生选项添加头像 + 姓名 + 科目
- 选中状态使用主色调高亮 + 对勾图标

**③ 添加按钮 (FAB)**
- 与学生管理页保持一致风格
- 颜色使用蓝色主题

**④ 模态框优化**
- 输入字段分组设计（学生选择、日期、课时，每组有标题）
- 自动计算金额区域使用独立的高亮卡片
- 金额数字使用较大字号 + 渐变色彩

### 4.4 统计页 (StatsScreen.tsx)

#### 改造要点：

**① 顶部统计卡片**
- 4 个统计卡片改为 2x2 网格，每个卡片带图标和数值
- 数值使用等宽数字效果
- 添加环形进度或彩色指示条

**② 收款概览区**
- 改为水平进度条形式展示已收/待收比例
- 已收金额和待收金额用不同颜色的大号数字展示
- 进度条使用渐变色填充

**③ 学生账单表格**
- 表头使用浅色毛玻璃效果固定在顶部
- 行与行之间使用浅色交替条纹（zebra striping）
- 待交金额 > 0 的行使用淡红色背景高亮
- 学生姓名列支持点击展开详情

**④ 空状态**
- 空数据时展示友好的引导信息

## 五、底部导航栏改造

### 改造要点：
- 背景使用白色 + 顶部细阴影（类似 iOS 风格）
- 中间添加按钮使用略微上浮的圆形设计（带渐变）
- 选中状态使用主色调 + 图标填充动画
- 标签文字使用更小字号
- 添加 `react-native-reanimated` 实现的切换动画（图标弹性效果）

## 六、动画与微交互方案

| 交互场景 | 动画效果 | 实现方式 |
|---------|---------|---------|
| 页面进入 | 内容从下往上淡入 (stagger) | `Animated.timing` / Reanimated |
| 列表项出现 | 依次从右滑入 + 淡入 | `FlatList` item animation |
| 按钮按下 | 缩放至 0.95 + 轻微阴影加深 | `Animated.spring` |
| 付款状态切换 | 弹跳动画 + 颜色渐变 | `LayoutAnimation` |
| FAB 按钮 | 持续呼吸脉冲 | `Animated.loop` |
| 数字变化 | 递增/递减计数动画 | `Animated.timing` (数字插值) |
| Tab 切换 | 图标弹性形变 | Reanimated `withSpring` |
| 模态框弹出 | 从底部滑入 + 背景渐暗 | `Animated.timing` |
| 删除操作 | 卡片向右滑出 + 淡出 | 手势 + `Animated.timing` |

### 实现优先级

1. **阶段一**：色彩体系 + 排版 + 阴影（全局样式重构）
2. **阶段二**：各页面布局改造（不改动业务逻辑）
3. **阶段三**：动画与微交互（Animated API）
4. **阶段四**：底部导航栏美化
5. **阶段五**：细节打磨（空状态、加载态、错误态）
